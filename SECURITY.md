# Security Policy

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report them privately by emailing **arielespinoza46@gmail.com** with the subject line `[tenancy-core] Security Vulnerability`.

Include:

- A clear description of the vulnerability
- Steps to reproduce or a proof-of-concept
- The version(s) affected
- Any suggested fix, if you have one

You will receive an acknowledgement within **72 hours**. We aim to release a fix within **14 days** of confirmation, depending on severity and complexity.

Once a fix is released, the vulnerability will be publicly disclosed in the [CHANGELOG](CHANGELOG.md).
